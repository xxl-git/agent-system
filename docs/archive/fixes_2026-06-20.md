# Agent System 遗留问题修复 (2026-06-20)

## 修复清单 (7个问题，6个代码修复)

### 1. 🔴 能力探测双超时竞争 (capability-probe.ts)
**问题**: `Promise.race([chatFn, 30s timeout])` 中的冗余 30s 超时与 SmartAdapter 的 30s `callWithTimeout` 完全重叠，但 `Promise.race` 不取消失败方。当 SmartAdapter 超时→内部 `localTimeouts`++→触发死循环降级→返回降级响应→Probe 收到降级响应→判定探测失败→Chain 收到 FAIL→降级。
**另案**: SmartAdapter 重试时 3×30s=90s → Probe 的 30s 超时会先触发 → 即使 SmartAdapter 还在尝试恢复也被 Probe 截断。
**修复**: 移除 Probe 层的 30s timeout，超时完全由 SmartAdapter 统一控制。
**文件**: `src/models/probe/capability-probe.ts` + `dist/models/probe/capability-probe.js`

### 2. 🔴 检查点任务卡死 (checkpoint.ts)
**问题**: 14 个旧检查点文件在 `data/checkpoints/` 中残留，`recoverySummary()` 反复报告"待恢复"，但恢复逻辑从未触发。每次心跳都扫描并显示。
**修复**: 新增 `clearAll()` 方法，在 `AgentCore.init()` 中自动清理上一轮会话的检查点（新会话的旧检查点无效）。
**文件**: `src/resilience/checkpoint.ts` → 新增 clearAll() 方法
`src/core/agent/agent-core.ts` → init() Step 2 清理检查点 + 步骤序号 X/7 修正
`dist/resilience/checkpoint.js` + `dist/core/agent/agent-core.js` 同步

### 3. 🟡 `(this.adapter as any)` 消除 (多个文件)
**问题**: 4 处使用 `(this.adapter as any)` 绕过 TypeScript 类型检查，调用 LMStudioAdapter 特有方法（`raw.contextLength`, `getEffectiveContextWindow()`, `markSessionReset()`, `isSessionReset()`）。
**修复**: 在 `SmartAdapter` 类中添加 5 个透明代理方法：
- `get contextLength()` → `this.raw.contextLength`
- `getEffectiveContextWindow()` → `this.raw.getEffectiveContextWindow()`
- `markSessionReset()` → `this.raw.markSessionReset()`
- `isSessionReset()` → `this.raw.isSessionReset()`  
- `clearSessionReset()` → `this.raw.clearSessionReset()`
**文件**: `src/core/smart-adapter.ts` + `dist/core/smart-adapter.js`
`src/core/agent/agent-core.ts` 中所有 `(adapter as any)` 替换为 `this.adapter.xxx()`

### 4. 🟡 LM Studio 空列表静默失败 (lmstudio.ts)
**问题**: `getCurrentModel()` 在 LM Studio 返回空 JSON 数组时直接返回原 `this.model`（可能为默认值 "lmstudio-community/..."），不输出任何日志或警告。
**修复**: 当模型列表为空时输出 `logger.warn`，区分「有历史名」和「无名」两种场景。
**文件**: `src/models/adapters/lmstudio.ts` + `dist/models/adapters/lmstudio.js`

### 5. 🟡 `safePath` 可被 URL 编码绕过 (agent-server.ts)
**问题**: 只检查 `input.includes('..')`，但 URL 编码 `%2e%2e` 不会被匹配到，且未检查 `~` 路径（Windows 下无意义但纯防御）和 Windows 绝对路径 `X:\`。
**修复**: 先 `decodeURIComponent()` 解码再检查 `..`/`~`/`X:`，增加二次校验（normalize 斜杠后比较）。
**文件**: `src/server/agent-server.ts` + `dist/server/agent-server.js`

### 6. 🟢 `ensureAgent()` 重复调用 `loadConfig()` (agent-server.ts)
**问题**: `loadConfig()` 在模块级已调用，又在 `ensureAgent()` 中再次调用。
**修复**: 移除 `ensureAgent()` 中的重复调用。
**文件**: `src/server/agent-server.ts`（dist 中已无此问题）

### 7. 🟢 init() 步骤序号修正
**问题**: "Step 3/6" 出现了两次（DB 和记忆恢复），Step 4/6 实际是第 5 步，总共 6 步但实际有 7 步。
**修复**: 重新编号为 Step 1/7 ~ Step 7/7，新增 Step 2/7（检查点清理）。
**文件**: `src/core/agent/agent-core.ts` + `dist/core/agent/agent-core.js`
