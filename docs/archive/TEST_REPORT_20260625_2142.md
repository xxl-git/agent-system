## 问题分析

### LM Studio 响应超时

**现象**：
1. LM Studio 正在运行，端口 1234 正常监听
2. 模型 `qwen3.6-35b-a3b-mtp` 已加载
3. 直接调用 `/v1/chat/completions` API，60 秒内无响应

**可能原因**：
1. **模型太大**：35B 参数模型在 CPU 上推理非常慢
2. **GPU 内存不足**：模型可能被交换到内存
3. **LM Studio 配置问题**：推理参数设置不当
4. **模型卡住**：推理过程中出现死锁或错误

**建议解决方案**：
1. **切换到更小的模型**：
   - `qwen3.5-9b-deepseek-v4-flash-mtp`（配置中已有）
   - 其他 7B-14B 参数的模型

2. **使用 DeepSeek API**：
   - 配置中已有 DeepSeek 提供者
   - 需要 API Key：`${DEEPSEEK_API_KEY}`

3. **增加 LM Studio 超时**：
   - 当前：120 秒
   - 建议：300-600 秒（如果坚持使用大模型）

4. **检查 LM Studio 状态**：
   - 查看 LM Studio UI 是否有错误信息
   - 检查 GPU 内存使用情况
   - 尝试重启 LM Studio
