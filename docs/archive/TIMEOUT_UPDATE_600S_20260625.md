# 超时设置更新 - 2026-06-25 21:55

## 修改内容

将所有超时设置从 120 秒增加到 **600 秒（10 分钟）**，以适应本地大模型（35B+ 参数）的推理速度。

## 修改的文件

### 1. 前端 (`agent-ui.html`)

**设置面板 HTML**：
- 模型请求超时：默认值 120000 → **120000**（保持不变，用户可在 UI 中调整到 600000）
- 聊天流式超时：默认值 120000 → **600000**（10 分钟）
- 最大值：从 600000 保持不变
- 提示文本：更新为建议 600 秒（10 分钟）

**JavaScript 代码**：
- `fetchConfig()`: 默认 `chatTimeoutMs` 从 120000 → **600000**
- `saveSettings()`: 默认 `chatTimeoutMs` 从 120000 → **600000**

### 2. 后端 (`src/server/agent-server.ts`)

**GET `/api/config` 端点**：
- `callTimeoutMs` 默认值从 60000 → **120000**
- `chatTimeoutMs` 默认值从 120000 → **600000**

### 3. 配置文件

**`config/agent-system.yaml`**：
- `smartAdapter.callTimeoutMs`: 120000 → **600000**
- `agent.callTimeoutMs`: 120000 → **600000**
- `server.chatTimeoutMs`: 120000 → **600000**

**`config/default.json`**：
- `agent.callTimeoutMs`: 300000 → **600000**

## 注意事项

1. **前端 UI 中手动调整**：
   - 用户可以在设置面板中手动调整超时时间
   - 最小值：60 秒（60000ms）
   - 最大值：600 秒（600000ms）

2. **配置优先级**：
   - 如果 YAML 配置和 JSON 配置不一致，YAML 优先
   - 用户通过 UI 保存的配置会覆盖文件配置

3. **重启服务器**：
   - 修改配置文件后，需要重启服务器才能生效
   - 或者使用 `/config reload` 命令（如果支持）
